**PROMPT:**

We need to implement a daily usage system for Shepherd’s Path that aligns with the app’s core experience: calm, non-transactional, and human.

This should NOT feel like a hard paywall or quota system. It should feel like a natural pause.

Requirements:

1. Daily Use Structure

* Set a soft limit of 5 meaningful AI interactions per user per day
* Allow a grace buffer of 1–2 additional uses (total possible: 6–7)
* Do NOT strictly enforce the limit at exactly 5
* System should allow flexibility rather than rigid cutoff

2. Never Interrupt Mid-Experience

* If a user begins an interaction, ALWAYS allow it to complete fully
* Never block, truncate, or cut off a response mid-flow
* Limits should only be evaluated AFTER a completed interaction

3. Tone & Messaging (CRITICAL)

* Do NOT use language like:
  “limit reached”
  “upgrade required”
  “you’ve used all your credits”

* Instead, use language that feels like a natural pause, such as:
  “That’s enough for today… take some time with what you’ve received.”
  “You don’t have to keep going right now.”
  “Come back when you’re ready.”

* Messaging should feel:
  calm
  respectful
  non-pressured
  spiritually aligned

4. UI Behavior

* Do NOT show a visible counter (no “3/5 uses remaining”)
* Do NOT create urgency or pressure
* Only surface messaging when the user attempts another interaction AFTER the soft limit + grace buffer

5. Unlock / Continuation (if applicable)

* If monetization is introduced, it should feel like an invitation, not a restriction
* Example tone:
  “If you’d like to keep going, you can continue here.”
* Avoid aggressive prompts or friction-heavy paywalls

6. System Design Notes

* Track daily usage per user (reset every 24 hours or at local midnight)
* Use soft logic rather than hard blocking wherever possible
* Prioritize experience over strict enforcement

Goal:
This system should feel like a thoughtful boundary, not a restriction. The user should never feel cut off — only gently guided to pause.
