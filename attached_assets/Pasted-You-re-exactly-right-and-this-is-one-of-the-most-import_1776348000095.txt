**PROMPT:**

We are building a faith-based application called Shepherd’s Path. In this product, UI/UX performance is not a secondary concern—it is the primary delivery system of the experience.

If the interface is slow, confusing, glitchy, or distracting—even briefly—the emotional and spiritual experience breaks immediately. This is not acceptable.

Treat UI/UX performance and fluidity as mission-critical.

---

## 1. CORE PRINCIPLE

Every interaction must feel:

* instant or near-instant
* smooth and uninterrupted
* visually calm and stable
* free of friction, lag, or confusion

The user should never notice the system. The experience should feel invisible.

---

## 2. NON-NEGOTIABLE PERFORMANCE STANDARDS

### A. Response Speed

* All primary interactions (tap → response) must occur within ~100–300ms perceived response time
* Audio (Listen feature) must start immediately (no noticeable delay)
* AI responses should show graceful loading states, never blank or frozen screens

---

### B. Zero Interruption Policy

* No UI freezing
* No delayed button response
* No jank, stutter, or layout shift
* No unexpected jumps in scroll position

If an interaction cannot be completed smoothly, it must be redesigned—not shipped.

---

### C. State Continuity

* Preserve scroll position at all times
* Preserve user context when navigating back
* Never reset the user to the top unless explicitly intended

---

### D. Audio Experience (CRITICAL)

* Audio playback must feel immediate and reliable
* No buffering delays
* No playback failures
* Clear loading indication if needed, but minimal

Audio is a core emotional feature and must feel seamless.

---

## 3. VISUAL CALM & STABILITY

* No flickering elements
* No rapid transitions
* No competing animations
* Maintain consistent spacing and layout

The UI should feel like a calm environment, not an active interface.

---

## 4. LOADING & TRANSITION STATES

Never leave the user in uncertainty.

Use soft, calm indicators such as:

* subtle fade-ins
* gentle placeholders
* minimal text like “Give me a moment…”

Avoid:

* spinners without context
* blank screens
* abrupt content jumps

---

## 5. ERROR HANDLING

Errors must never feel technical or disruptive.

Instead of:
“Something went wrong”

Use:
“We can try that again.”

Maintain tone:

* calm
* human
* non-technical

---

## 6. PRIORITY ORDER

When making implementation decisions, always prioritize:

1. Stability
2. Speed
3. Simplicity
4. Visual calm

Over:

* feature density
* complexity
* novelty

---

## 7. FAILURE CONDITIONS (DO NOT SHIP IF PRESENT)

* Any noticeable lag on primary actions
* Audio delay or inconsistency
* UI elements that require multiple taps
* Confusing navigation or lost position
* Any moment where the user might hesitate due to system behavior

---

## 8. SUCCESS CRITERIA

The experience is successful if:

* The user never thinks about the interface
* There are no moments of hesitation or confusion
* The flow feels continuous and uninterrupted
* The user stays emotionally engaged without distraction

---

## FINAL DIRECTIVE

Do not treat this as a typical app UX.

This is a reflective, emotionally sensitive environment.

Any friction, delay, or instability breaks trust immediately.

Optimize for invisibility, fluidity, and calm above all else.
